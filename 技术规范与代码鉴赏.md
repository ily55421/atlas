## 一、架构设计规范

### 1.1 分层架构

Apache Atlas 采用**清晰的四层架构**，各层职责明确：

```
┌─────────────────────────────────────────┐
│         Web Layer (Filter/Servlet)      │  ← 安全过滤、认证、审计
├─────────────────────────────────────────┤
│         API Layer (REST Resources)      │  ← HTTP 端点、参数校验
├─────────────────────────────────────────┤
│      Business Layer (Services/Store)    │  ← 业务逻辑、事务管理
├─────────────────────────────────────────┤
│      Data Layer (Repository/GraphDB)    │  ← 数据持久化、索引
└─────────────────────────────────────────┘
```

**规范要点**：
- ✅ 层与层之间通过接口交互，禁止跨层调用
- ✅ 每层有明确的包结构（`filters`、`rest`、`service`、`repository`）
- ✅ 依赖方向：Web → API → Business → Data，禁止反向依赖

### 1.2 模块化设计

项目采用 **Maven 多模块**架构，模块职责清晰：

| 模块 | 职责 | 依赖方向 |
|------|------|---------|
| `apache-atlas` (根 POM) | 统一依赖版本管理 | 无 |
| `atlas-common` | 公共工具类、自定义注解 | 无 |
| `atlas-intg` | 核心异常、枚举、接口定义 | common |
| `atlas-client` | 客户端 SDK | intg |
| `atlas-repository` | 元数据存储库实现 | intg, graphdb |
| `atlas-authorization` | 授权接口与实现 | intg |
| `atlas-webapp` | Web 应用入口 | repository, authorization, client |
| `addons/*-bridge` | 外部系统集成 | intg, client |

**规范要点**：
- ✅ 核心模块（intg、common）不依赖业务模块
- ✅ 业务模块通过接口依赖，而非实现依赖
- ✅ 插件化设计：桥接器（bridge）作为独立模块，按需加载

## 二、代码编写规范

### 2.1 命名规范

| 元素 | 规范 | 示例 |

### 2.3 代码组织规范

**包结构规范**（以 webapp 模块为例）：
```
org.apache.atlas.web
  ├── dao/           # 数据访问层接口与实现
  ├── errors/        # 异常映射器（ExceptionMapper）
  ├── filters/       # Servlet 过滤器链
```

**规范要点**：
- ✅ 按功能分包，而非按技术分包
- ✅ 每个包内职责单一，高内聚低耦合
- ✅ 禁止循环依赖

## 三、API 设计规范

### 3.1 RESTful 设计规范

**端点设计**：
```
GET    /api/atlas/v2/entities/{guid}              # 获取单个实体
POST   /api/atlas/v2/entities                     # 创建实体
PUT    /api/atlas/v2/entities                     # 更新实体
DELETE /api/atlas/v2/entities?guid={guid}         # 删除实体
GET    /api/atlas/v2/entities/unique/attribute    # 按属性查询
GET    /api/atlas/v2/entities/{guid}/lineage      # 获取血缘关系
```

**规范要点**：
- ✅ 使用名词复数形式（`/entities` 而非 `/entity`）
- ✅ 使用 HTTP 方法表达操作意图
- ✅ 使用查询参数表达过滤和分页
- ✅ 嵌套资源使用子路径（`/{guid}/lineage`）

### 3.2 请求响应规范

**请求处理**：
```java
@POST
@Path("/")
@Consumes(Servlets.JSON_MEDIA_TYPE)
@Produces(Servlets.JSON_MEDIA_TYPE)
@Timed
public AtlasEntityWithExtInfo createEntity(AtlasEntityWithExtInfo entity) throws AtlasBaseException {
    // 参数校验
    if (entity == null) {
        throw new AtlasBaseException(AtlasErrorCode.INVALID_PARAMETERS);
    }
    
    // 业务逻辑
    return entitiesStore.create(entity);
}
```

**规范要点**：
- ✅ 使用 `@Consumes` / `@Produces` 明确指定媒体类型
- ✅ 使用 `@Timed` 进行性能监控
- ✅ 入参校验，抛出明确的异常
- ✅ 返回统一的响应格式

### 3.3 版本管理规范

项目同时维护 **v1** 和 **v2** 两套 API：

| 版本 | 包路径 | 端点前缀 | 状态 |
|------|--------|---------|------|
| **v1** | `org.apache.atlas.web.resources` | `/api/atlas/v1/` | 维护模式，不再新增功能 |
| **v2** | `org.apache.atlas.web.rest` | `/api/atlas/v2/` | 活跃开发，推荐使用 |

**规范要点**：
- ✅ 新版本 API 使用独立包路径
- ✅ 向后兼容旧版本 API
- ⚠️ 建议在文档中明确 v1 的 deprecation 时间线

## 四、安全开发规范

### 4.1 认证机制

项目支持**多种认证方式**，通过配置切换：

| 认证方式 | 配置项 | 实现类 |
|---------|--------|--------|
| **文件认证** | `atlas.authentication.method.file=true` | `AtlasFileAuthenticationProvider` |
| **LDAP 认证** | `atlas.authentication.method.ldap=true` | `AtlasLdapAuthenticationProvider` |
| **AD 认证** | `atlas.authentication.method.ad=true` | `AtlasADAuthenticationProvider` |
| **Keycloak** | `atlas.authentication.method.keycloak=true` | `AtlasKeycloakAuthenticationProvider` |
| **Kerberos** | `atlas.authentication.method.kerberos=true` | `AtlasAuthenticationFilter` |
| **PAM 认证** | `atlas.authentication.method.pam=true` | `AtlasPamAuthenticationProvider` |

**安全配置模板**：
```java
@Configuration
@EnableWebSecurity
@EnableGlobalMethodSecurity(prePostEnabled = true)
public class AtlasSecurityConfig extends WebSecurityConfigurerAdapter {
    
    @Inject
    private AtlasAuthenticationProvider authenticationProvider;
    
    @Inject
    private AtlasAuthenticationFilter authenticationFilter;
    
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http.csrf().disable()
            .authorizeRequests()
            .antMatchers("/api/atlas/admin/**").hasRole("ADMIN")
            .anyRequest().authenticated()
            .and()
            .addFilterBefore(authenticationFilter, BasicAuthenticationFilter.class);
    }
}
```

### 4.2 CSRF 防护

**实现位置**：`AtlasCSRFPreventionFilter`

**防护策略**：
1. 检查请求头中的 `X-XSRF-Token` 或 `X-XSRF-HEADER`
2. 支持配置忽略的 HTTP 方法（如 GET、HEAD、OPTIONS）
3. 支持配置可信浏览器 User-Agent 正则匹配
4. 使用策略模式（`HttpInteraction` 接口）支持不同的响应处理方式

**规范要点**：
- ✅ 所有修改数据的请求（POST/PUT/DELETE）必须携带 CSRF Token
- ✅ 敏感操作必须进行 CSRF 校验
- ✅ 允许配置白名单方法减少误报

### 4.3 授权机制

**核心接口**：`AtlasAuthorizer`

```java
public interface AtlasAuthorizer {
    void init();
    void cleanUp();
    
    boolean isAccessAllowed(AtlasAdminAccessRequest request) throws AtlasAuthorizationException;
    boolean isAccessAllowed(AtlasEntityAccessRequest request) throws AtlasAuthorizationException;
    boolean isAccessAllowed(AtlasTypeAccessRequest request) throws AtlasAuthorizationException;
    
    default boolean isAccessAllowed(AtlasRelationshipAccessRequest request) throws AtlasAuthorizationException {
        return true;
    }
    
    default void scrubSearchResults(AtlasSearchResultScrubRequest request) throws AtlasAuthorizationException {}
    default void filterTypesDef(AtlasTypesDefFilterRequest request) throws AtlasAuthorizationException {}
}
```

**设计亮点**：
- ✅ 按访问类型分离接口（Admin、Entity、Type、Relationship）
- ✅ 使用 Java 8 `default` 方法提供默认实现，向后兼容
- ✅ 支持三种授权器：`SIMPLE`（内置）、`RANGER`（Apache Ranger）、`NONE`（无授权）

### 5.2 错误码规范

**位置**：`AtlasErrorCode.java`（枚举类）

### 5.3 异常映射器（ExceptionMapper）

**实现位置**：`webapp/src/main/java/org/apache/atlas/web/errors/`

| 映射器 | 处理异常 | HTTP 状态码 |
|--------|---------|------------|
| `AtlasBaseExceptionMapper` | `AtlasBaseException` | 根据错误码动态返回 |
| `NotFoundExceptionMapper` | `NotFoundException` | 404 |
| `AllExceptionMapper` | 所有其他异常 | 500 |

**异常映射器模板**：
```java
@Provider
@Component
public class NotFoundExceptionMapper implements ExceptionMapper<NotFoundException> {
    
    private static final Logger LOG = LoggerFactory.getLogger(NotFoundExceptionMapper.class);
    
    @Override
    public Response toResponse(NotFoundException exception) {
        LOG.error("NotFoundException caught", exception);
        
        AtlasError error = new AtlasError(
            "ATLAS-404-00-001",
            exception.getMessage(),
            "Resource not found"
        );
        
        return Response.status(Response.Status.NOT_FOUND)
                       .entity(error)
                       .type(MediaType.APPLICATION_JSON)
                       .build();
    }
}
```

**规范要点**：
- ✅ 使用 `@Provider` + `@Component` 注解
- ✅ 实现 `ExceptionMapper<T>` 接口
- ✅ 统一日志格式和错误响应结构
- ✅ 为每个请求生成唯一 ID 用于日志追踪

## 六、日志与可观测性规范

### 6.1 日志框架

**技术栈**：SLF4J + Logback

**日志级别使用规范**：

| 级别 | 使用场景 | 示例 |
|------|---------|------|
| **ERROR** | 系统错误、异常 | `LOG.error("Failed to create entity", e)` |
| **WARN** | 警告信息、降级 | `LOG.warn("Entity not found, returning null")` |
| **INFO** | 关键业务流程 | `LOG.info("==> EntityREST.getById({})", guid)` |
| **DEBUG** | 调试信息 | `LOG.debug("Processing entity: {}", entity.toJson())` |
| **TRACE** | 详细追踪 | `LOG.trace("Request headers: {}", headers)` |

### 6.2 日志命名规范

| 日志变量 | 用途 | 示例 |
|---------|------|------|
| `LOG` | 通用业务日志 | `private static final Logger LOG = LoggerFactory.getLogger(EntityREST.class)` |
| `PERF_LOG` | 性能监控日志 | `private static final Logger PERF_LOG = AtlasPerfTracer.getPerfLogger("perf.log.rest")` |
| `AUDIT_LOG` | 审计日志 | `private static final Logger AUDIT_LOG = LoggerFactory.getLogger("AUDIT")` |

### 6.3 日志最佳实践

**✅ 正确做法**：
```java
// 1. 使用参数化日志（避免不必要的字符串拼接）
LOG.info("==> EntityREST.getById({})", guid);

// 2. 使用 isDebugEnabled() 检查
if (LOG.isDebugEnabled()) {
    LOG.debug("Processing complex entity: {}", entity.getComplexJson());
}

// 3. 入口/出口使用 ==> 和 <== 标记
LOG.info("==> EntityREST.createEntity()");
// ... 业务逻辑 ...
LOG.info("<== EntityREST.createEntity()");
```

**❌ 错误做法**：
```java
// 1. 字符串拼接（即使不打印也会执行拼接）
LOG.info("==> EntityREST.getById(" + guid + ")");

// 2. 未检查级别直接调用
LOG.debug("Complex entity data: " + entity.getComplexJson());
```

### 6.4 性能监控

**自定义注解**：`@Timed`

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Timed {
    String value() default "";
}
```

**使用模式**：
```java
@Timed
public AtlasEntityWithExtInfo getById(String guid) throws AtlasBaseException {
    AtlasPerfTracer perf = null;
    try {
        if (AtlasPerfTracer.isPerfTraceEnabled(PERF_LOG)) {
            perf = AtlasPerfTracer.getPerfTracer(
                PERF_LOG, 
                "EntityREST.getById(" + guid + ")"
            );
        }
        return entitiesStore.getById(guid);
    } finally {
        AtlasPerfTracer.log(perf);
    }
}
```

**规范要点**：
- ✅ 所有 REST 端点使用 `@Timed` 注解
- ✅ 使用 try-finally 确保性能日志被记录
- ✅ 使用 `isPerfTraceEnabled()` 避免不必要的对象创建
- ⚠️ 建议后续使用 AOP 自动化此过程

### 6.5 请求追踪

**实现位置**：`AuditFilter`

```java
public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) 
        throws IOException, ServletException {
    
    // 为每个请求生成唯一 ID
    String requestId = UUID.randomUUID().toString();
    
    // 将 requestId 加入线程名，便于日志追踪
    String origThreadName = Thread.currentThread().getName();
    Thread.currentThread().setName(origThreadName + "-requestId-" + requestId);
    
    try {
        long startTime = System.currentTimeMillis();
        
        // 执行请求
        chain.doFilter(request, response);
        
        long endTime = System.currentTimeMillis();
        
        // 记录审计日志
        AUDIT_LOG.info("requestId={}, method={}, uri={}, user={}, ip={}, duration={}ms",
                      requestId, method, uri, user, ip, (endTime - startTime));
    } finally {
        Thread.currentThread().setName(origThreadName);
    }
}
```

**规范要点**：
- ✅ 每个请求必须有唯一 ID
- ✅ 请求 ID 必须包含在线程名中
- ✅ 审计日志记录关键信息（用户、IP、耗时）

## 七、设计模式应用

### 7.1 工厂模式

**应用位置**：`AtlasAuthorizerFactory`

```java
public class AtlasAuthorizerFactory {
    private static volatile AtlasAuthorizerFactory instance;
    
    private AtlasAuthorizerFactory() {}
    
    public static AtlasAuthorizerFactory getInstance() {
        if (instance == null) {
            synchronized (AtlasAuthorizerFactory.class) {
                if (instance == null) {
                    instance = new AtlasAuthorizerFactory();
                }
            }
        }
        return instance;
    }
    
    public AtlasAuthorizer createAuthorizer(String authorizerImpl) {
        try {
            Class<?> clazz = Class.forName(authorizerImpl);
            return (AtlasAuthorizer) clazz.getDeclaredConstructor().newInstance();
        } catch (Exception e) {
            throw new AtlasAuthorizationException("Failed to create authorizer: " + authorizerImpl);
        }
    }
}
```

**设计亮点**：
- ✅ 双重检查锁定（DCL）实现线程安全单例
- ✅ 使用反射动态创建授权器实例
- ✅ 支持配置化选择授权器实现

### 7.2 策略模式

**应用位置**：`AtlasCSRFPreventionFilter.HttpInteraction`

```java
// 策略接口
public interface HttpInteraction {
    void sendForbidden(HttpServletResponse response) throws IOException;
}

// 策略实现
public class ServletFilterHttpInteraction implements HttpInteraction {
    @Override
    public void sendForbidden(HttpServletResponse response) throws IOException {
        response.sendError(HttpServletResponse.SC_FORBIDDEN, "CSRF verification failed");
    }
}

// 使用策略
public class AtlasCSRFPreventionFilter implements Filter {
    private HttpInteraction interaction;
    
    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
        this.interaction = new ServletFilterHttpInteraction();

```

**设计亮点**：
- ✅ 策略可替换，支持不同的响应处理方式
- ✅ 符合开闭原则（对扩展开放，对修改关闭）

### 7.3 模板方法模式

**应用位置**：`AtlasAbstractAuthenticationProvider`

```java
// 抽象基类
public abstract class AtlasAbstractAuthenticationProvider implements AuthenticationProvider {
    
    // 模板方法
    @Override
    public Authentication authenticate(Authentication authentication) throws AuthenticationException {
        // 1. 通用逻辑：获取用户信息
        UserGroupInformation ugi = getUserGroupInformation(authentication.getName());
        
        // 2. 通用逻辑：获取用户组
        Set<GrantedAuthority> authorities = getAuthoritiesFromUGI(ugi);
        
        // 3. 子类实现：具体认证逻辑
        authenticateUser(authentication, ugi);
        
        // 4. 通用逻辑：返回认证结果
        return new UsernamePasswordAuthenticationToken(
            authentication.getName(), 
            authentication.getCredentials(),
            authorities
        );
    }
    
    // 抽象方法，由子类实现
    protected abstract void authenticateUser(Authentication authentication, UserGroupInformation ugi)
```

### 8.3 鉴赏三：审计日志过滤器

**文件**：`webapp/src/main/java/org/apache/atlas/web/filters/AuditFilter.java`

```java
@Component
public class AuditFilter implements Filter {
    private static final Logger LOG = LoggerFactory.getLogger(AuditFilter.class);
    private static final Logger AUDIT_LOG = LoggerFactory.getLogger("AUDIT");
    
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws IOException, ServletException {
        
        HttpServletRequest httpRequest = (HttpServletRequest) request;
        HttpServletResponse httpResponse = (HttpServletResponse) response;
        
        // 1. 生成请求唯一标识
        String requestId = UUID.randomUUID().toString();
        
        // 2. 将 requestId 加入线程名，便于日志追踪
        String origThreadName = Thread.currentThread().getName();
        Thread.currentThread().setName(origThreadName + "-requestId-" + requestId);
        
        // 3. 记录请求信息
        String method = httpRequest.getMethod();
        String uri = httpRequest.getRequestURI();
        String user = httpRequest.getRemoteUser();
        String ip = httpRequest.getRemoteAddr();
        
        long startTime = System.currentTimeMillis();
        
        try {
            // 4. 执行请求
            chain.doFilter(httpRequest, httpResponse);
            
            long endTime = System.currentTimeMillis();
            long duration = endTime - startTime;
            
            // 5. 记录审计日志
            AUDIT_LOG.info("requestId={}, method={}, uri={}, user={}, ip={}, status={}, duration={}ms",
                          requestId, method, uri, user, ip, httpResponse.getStatus(), duration);
            
        } catch (Exception e) {
            long endTime = System.currentTimeMillis();
            long duration = endTime - startTime;
            
            // 6. 记录异常
            AUDIT_LOG.error("requestId={}, method={}, uri={}, user={}, ip={}, error={}, duration={}ms",
                           requestId, method, uri, user, ip, e.getMessage(), duration, e);
            throw e;
        } finally {
            // 7. 恢复原始线程名
            Thread.currentThread().setName(origThreadName);
        }
    }
}
```

**鉴赏要点**：
1. ✅ **请求追踪**：每个请求生成唯一 UUID，全链路追踪
2. ✅ **线程名注入**：将 requestId 加入线程名，所有日志自动携带追踪信息
3. ✅ **完整记录**：记录方法、URI、用户、IP、状态码、耗时等关键信息
4. ✅ **异常处理**：捕获异常并记录，不吞异常
5. ✅ **资源清理**：在 finally 块中恢复线程名，避免线程池复用问题
6. ✅ **性能友好**：使用参数化日志，避免不必要的字符串拼接

```java
@Component
public class AtlasCSRFPreventionFilter implements Filter {
    private static final Logger LOG = LoggerFactory.getLogger(AtlasCSRFPreventionFilter.class);
    
    // CSRF Token 头部名称
    private String csrfHeaderName = "X-XSRF-Token";
    
    // 忽略的 HTTP 方法
    private Set<String> ignoreMethods = new HashSet<>(Arrays.asList("GET", "HEAD", "OPTIONS"));
    
    // 策略接口
    private HttpInteraction interaction;
    
    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
        // 初始化策略实现
        this.interaction = new ServletFilterHttpInteraction();
        
        // 从配置读取忽略方法
        String ignoreMethodsStr = filterConfig.getInitParameter("ignoreMethods");
        if (ignoreMethodsStr != null) {
            this.ignoreMethods = new HashSet<>(Arrays.asList(ignoreMethodsStr.split(",")));
        }
    }
    
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws IOException, ServletException {
        
        HttpServletRequest httpRequest = (HttpServletRequest) request;
        HttpServletResponse httpResponse = (HttpServletResponse) response;
        
        String method = httpRequest.getMethod();
        
        // 1. 检查是否为忽略的方法
        if (ignoreMethods.contains(method.toUpperCase())) {
            chain.doFilter(request, response);
            return;
        }
        
        // 2. 检查 CSRF Token
        String csrfToken = httpRequest.getHeader(csrfHeaderName);
        if (csrfToken == null || csrfToken.isEmpty()) {
            LOG.warn("CSRF token missing in request: {} {}", method, httpRequest.getRequestURI());
            interaction.sendForbidden(httpResponse);
            return;
        }
        
        // 3. Token 验证逻辑（简化版）
        if (!validateCsrfToken(csrfToken)) {
            LOG.warn("CSRF token validation failed: {} {}", method, httpRequest.getRequestURI());
            interaction.sendForbidden(httpResponse);
            return;
        }
        
        // 4. 验证通过，继续过滤器链
        chain.doFilter(request, response);
    }
    
    /**
     * 验证 CSRF Token
     */
    private boolean validateCsrfToken(String token) {
        // Token 验证逻辑
        return true;
    }
}

/**
 * HTTP 交互策略接口
 */
public interface HttpInteraction {
    void sendForbidden(HttpServletResponse response) throws IOException;
}

/**
 * Servlet 过滤器实现
 */

    @Override
    public void sendForbidden(HttpServletResponse response) throws IOException {
        response.sendError(HttpServletResponse.SC_FORBIDDEN, "CSRF verification failed");
    }
```

1. ✅ **策略模式**：使用 `HttpInteraction` 接口抽象响应处理，可替换
2. ✅ **配置化**：忽略方法从配置读取，无需修改代码
